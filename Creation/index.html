Что такое Code Style и зачем он нужен?

Code style (он же coding standards, coding convention или programming style) — набор правил и соглашений для написания кода, над которым работает более одного разработчика.

Зачем нужны правила?

  • Убрать разногласия между разработчиками, так как каждый привык писать по своему и считать свой код самым правильным
  • Предотвратить появление распространённых ошибок в коде. Например, then() написали, а catch() забыли, и в итоге возникла ошибка на проде.
  • Упростить чтение кода коллегами-разработчиками

Где найти правила?

Для того, чтобы дорогой читатель не бегал по всемирной паутине в поисках описания правил, и не тратил свои драгоценные время и нервные клетки, а так же мы приготовили две замечательные ссылки.
